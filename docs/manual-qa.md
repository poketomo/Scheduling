# Scheduling MVP 手動QA

## ローカルサーバー起動方法

```powershell
cd C:\Users\pokek\Documents\Scheduling
python -m http.server 8123 --bind 127.0.0.1
```

ブラウザで [http://127.0.0.1:8123/index.html](http://127.0.0.1:8123/index.html) を開く。

## サンプルデータ投入方法

1. 画面右上の `サンプル投入` を押す
2. 講師・生徒・可能時間・受講希望が投入される
3. サイドバーの件数が増えることを確認する

## 講師追加の確認方法

1. `講師一覧・追加編集` タブを開く
2. `追加` を押す
3. 名前、性別、対応教科、可能時間を入力して `保存`
4. 左側一覧に新しい講師が出ることを確認する

## 生徒追加の確認方法

1. `生徒一覧・追加編集` タブを開く
2. `追加` を押す
3. 名前、手のかかる度、希望教科、可能時間を入力して `保存`
4. 左側一覧に新しい生徒が出ることを確認する

## 1人の生徒に複数教科を設定する

1. `生徒一覧・追加編集` タブを開く
2. 任意の生徒で `希望教科` を2つ以上チェックする
3. `受講希望設定` に教科ごとの行が増えることを確認する

## 週2回希望を設定する

1. 生徒編集の `受講希望設定` で対象教科の `週回数` を `2` にする
2. `保存` を押す
3. `自動生成` 後、同一生徒の同一教科が2件生成されることを確認する

## NG講師を設定する

1. 生徒編集の `希望しない先生` で講師を選ぶ
2. `保存` 後に `自動生成` を実行する
3. 結果にその講師が出ないことを確認する

## 時間割作成の確認方法

1. `時間割設定` タブを開く
2. `スロット追加` を押す
3. 曜日、開始時刻、終了時刻、表示ラベルを入力して保存する
4. 一覧に追加されたことを確認する

## グリッド操作の確認方法

1. 講師または生徒の編集画面で `可能時間` を見る
2. 1セルクリックで ON/OFF できることを確認する
3. マウスドラッグで複数セルをまとめて切り替えられることを確認する
4. 曜日ボタン、時間帯ボタン、`平日夜`、`土日` が効くことを確認する
5. `前回入力からコピー` と他人コピーが効くことを確認する

## 自動生成の確認方法

1. サンプル投入後に `自動生成` タブを開く
2. 入力不足の警告がないことを確認する
3. `作成する` を押す
4. `生成結果確認` タブへ遷移し、候補が表示されることを確認する

## 複数理由の未割当を確認する

1. サンプルデータの `森未割当` を含む候補を確認する
2. `未割当生徒` の理由欄にコード付きで複数行出せることを確認する

## 固定・再生成の確認方法

1. 生成結果の任意の割当にある `固定` を ON にする
2. `固定を維持して再生成` を押す
3. 同じ受講希望の割当が維持されることを確認する

## 確定の確認方法

1. 任意の候補で `候補案を確定` を押す
2. 候補一覧に `確定済み` が表示されることを確認する
3. `localStorage` の `scheduling-mvp-db-v1` を見て `confirmedAssignments` が増えることを確認する

## 確定済み割当が次回生成で衝突しないことを確認する

1. 候補を確定する
2. 再度 `作成する` または `固定を維持して再生成` を押す
3. 確定済み授業と同じ生徒・同じ時間帯の新規候補が重複しないことを確認する

## localStorageリセット方法

1. 画面右上の `データ初期化` を押す
2. ブラウザの DevTools から `localStorage` を開き、`scheduling-mvp-db-v1` が消えていることを確認する
